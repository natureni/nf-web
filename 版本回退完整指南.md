## 🔄 三种回退方法

### 方法1：软回退（最安全 - 推荐）

**特点**：保留所有更改，只回退提交历史

```bash
# 回退到上一个版本
git reset --soft HEAD~1

# 回退到指定版本
git reset --soft f570c74

# 回退到指定数量的版本前
git reset --soft HEAD~3
```

**适用场景**：
- 想要修改提交信息
- 想要重新组织提交
- 不想丢失任何代码更改

### 方法2：混合回退（默认方式）

**特点**：保留工作区更改，清空暂存区

```bash
# 回退到上一个版本
git reset HEAD~1

# 回退到指定版本
git reset f570c74
```

**适用场景**：
- 想要重新添加文件到暂存区
- 保留代码更改但重新组织提交

### 方法3：硬回退（危险 - 慎用）

**特点**：完全回退，丢失所有更改

```bash
# ⚠️ 危险：完全回退到上一个版本
git reset --hard HEAD~1

# ⚠️ 危险：完全回退到指定版本
git reset --hard f570c74
```

**适用场景**：
- 确定要丢失所有未提交的更改
- 想要完全回到历史状态

## 🚀 高级回退技巧

### 1. 撤销特定提交（推荐）

```bash
# 创建新提交来撤销指定提交
git revert 1eb2f18

# 撤销最近的提交
git revert HEAD

# 撤销多个提交
git revert HEAD~2..HEAD
```

### 2. 交互式回退

```bash
# 交互式重新排列最近3个提交
git rebase -i HEAD~3
```

### 3. 回退特定文件

```bash

# 将文件回退到上一个版本
git checkout HEAD~1 -- src/components/UserProfile.tsx
```

## 🛠️ 创建回退脚本

### 安全回退脚本
```batch
@echo off
echo 🔄 安全版本回退工具
echo ====================
echo.

echo 📜 当前版本历史：
git log --oneline -5
echo.

set /p version="请输入要回退到的版本号（如：f570c74）: "
set /p type="选择回退类型 (1=软回退, 2=混合回退, 3=硬回退): "

if "%type%"=="1" (
    echo 执行软回退...
    git reset --soft %version%
    echo ✅ 软回退完成，更改保留在暂存区
)

if "%type%"=="2" (
    echo 执行混合回退...
    git reset %version%
    echo ✅ 混合回退完成，更改保留在工作区
)

if "%type%"=="3" (
    echo ⚠️ 警告：这将丢失所有未提交的更改！
    set /p confirm="确定要继续吗？(y/N): "
    if /i "%confirm%"=="y" (
        git reset --hard %version%
        echo ✅ 硬回退完成
    ) else (
        echo 操作已取消
    )
)

echo.
echo 📊 回退后状态：
git log --oneline -3
echo.
pause
```

## 🔧 恢复操作

### 如果回退错了怎么办？

```bash
# 1. 查看操作历史
git reflog

# 2. 恢复到之前的状态
git reset --hard HEAD@{1}

# 3. 或者恢复到指定的reflog
git reset --hard HEAD@{2}
```

### 恢复暂存的更改

```bash
# 恢复最近的stash
git stash pop

# 查看所有stash
git stash list

# 恢复指定的stash
git stash apply stash@{1}
```

## ⚠️ 重要注意事项

### 1. 回退已推送的提交

```bash
# 如果已经推送到远程，需要强制推送
git push origin 20250708 --force

# 更安全的方式：使用force-with-lease
git push origin 20250708 --force-with-lease
```

### 2. 团队协作注意事项

- 回退已推送的提交前要通知团队
- 考虑使用`git revert`而不是`git reset`
- 重要版本回退前先创建备份分支

### 3. 数据安全

- 重要操作前先备份
- 使用`git stash`保存临时更改
- 定期创建标签标记重要版本
